ワーカー並列のため、Playwright の `workerInfo.workerIndex` (0..3) で DB を切り替える。worker N は `geo_base_e2e_wN` を使う。API 起動時の `DATABASE_URL` は単一だが、`POST /api/test/reset?worker=N` のように worker index を渡して該当 DB を truncate する設計に拡張する。

**重要な設計判断**:
- 単一の API プロセスで複数 DB を扱うには、reset endpoint が「どの DB を truncate するか」を知る必要がある。`worker` クエリパラメータを受け取って、`postgresql://...:5432/geo_base_e2e_wN` という connection string を組み立てて新規接続する形にする。
- 通常の API 操作（factory のシード）は `DATABASE_URL` (= `geo_base_e2e_w0` 等) を直接使う。worker が違うテストは異なる API プロセスではなく、同じ API プロセスが異なる DB に接続する形では破綻する → **API も worker 別に起動する必要がある**。
- 実装簡略化のため Phase 2 は **worker 数 4 で API を 4 並列起動**しない方針にする：代わりに「**並列度を `fullyParallel: true` のまま、テストファイル単位で順次実行（workers: 1）**」を維持しつつ、テストファイルを増やすことに注力する。
- 並列実行 (workers: 4) は別途 Phase 2.5 / Phase 3 で再評価する。本 Phase は core テスト網羅と CI 拡充に絞る。

**この Task 1 は「ワーカー別 DB の基盤コード」のみ実装し、Playwright 設定は workers: 1 のまま据え置く** — 将来の並列化のための土台のみを敷く。

- [ ] **Step 1: 98_create_e2e_db.sh をワーカー DB を含めて作成するように拡張**

`docker/postgis-init/98_create_e2e_db.sh` の中身を以下のように変更:

```bash
#!/bin/bash
# Create geo_base_e2e (worker 0) and optional worker DBs (geo_base_e2e_w0..w3).
# Issue #111: Phase 2 でワーカー並列化の足場を作る（実際の workers: 4 化は
# 後段で実施）。E2E_WORKER_COUNT で並列度を指定（デフォルト 1 = 単一 DB）。
set -euo pipefail

DEV_DB="${POSTGRES_DB:-geo_base}"
WORKER_COUNT="${E2E_WORKER_COUNT:-1}"

# Phase 1 互換: geo_base_e2e は常に作る。
DBS=("geo_base_e2e")

# Phase 2: w0..w(N-1) を追加で作る（WORKER_COUNT >= 1 の時）。
for i in $(seq 0 $((WORKER_COUNT - 1))); do
  DBS+=("geo_base_e2e_w${i}")
done

for DB in "${DBS[@]}"; do
  echo "[init] Creating E2E database: ${DB}"
  psql -v ON_ERROR_STOP=1 -U "${POSTGRES_USER}" -d postgres <<-EOSQL
    SELECT 'CREATE DATABASE ${DB}'
    WHERE NOT EXISTS (SELECT FROM pg_database WHERE datname = '${DB}')\gexec
EOSQL
  echo "[init] Cloning schema from ${DEV_DB} to ${DB}"
  pg_dump -U "${POSTGRES_USER}" --schema-only --no-owner --no-acl "${DEV_DB}" \
    | psql -v ON_ERROR_STOP=1 -U "${POSTGRES_USER}" -d "${DB}"
  echo "[init] E2E database ready: ${DB}"
done
```

```yaml
      shell: bash

        PGPORT: ${{ inputs.postgres-port }}
        PGUSER: postgres
        PGPASSWORD: postgres

            echo "Applying $f to $DB"
```

```python
from typing import Literal


@router.get("/tokens")
def get_recent_token(
    type: Literal["team_invitation"],
    email: str | None = None,
):
    """E2E から最新の token を引き取るための専用エンドポイント。

    現状サポート:
    - `type=team_invitation`: `team_invitations.token` (plain) を email で検索

    NOTE: `password_reset` は `password_reset_tokens.token_hash` しか保存
    されない仕様で平文を取り出せないため非対応。Phase 3 で console email
    backend のキャプチャ等の追加機構と合わせて対応する。
    """
    _assert_e2e_mode_or_die()
    _assert_e2e_database_or_die()
    if not email:
        raise HTTPException(status_code=400, detail="email is required")

    with get_db_connection() as conn:
        with conn.cursor() as cur:
            # status='pending' AND token IS NOT NULL で「有効な token 行」
            # のみに限定する。expired/cancelled で token=NULL の行が
            # accepted_at IS NULL のままヒットしないように。
            cur.execute(
                """
                SELECT token FROM team_invitations
                WHERE email = %s
                  AND status = 'pending'
                  AND token IS NOT NULL
                  AND accepted_at IS NULL
                ORDER BY created_at DESC LIMIT 1
                """,
                (email,),
            )
            row = cur.fetchone()
    if not row or not row[0]:
        raise HTTPException(
            status_code=404, detail=f"No {type} token for {email}"
        )
    return {"token": row[0]}
```

注意: スキーマのカラム名（`team_invitations.email` / `token` / `status` / `created_at` / `accepted_at`）は `docker/postgis-init/05_teams_schema.sql` を grep して実際のカラム名に合わせる。違う場合は SQL を調整。

```typescript
/**
 * E2E 専用: 直前に発行された password_reset / team_invitation の token を
 * API の test endpoint 経由で取得する。実 email 配信を待たずにテストが進められる。
 */
import { request } from "@playwright/test";

const API_BASE = process.env.PLAYWRIGHT_API_BASE_URL || "http://localhost:8000";

export type TokenType = "password_reset" | "team_invitation";

export async function fetchRecentToken(
  type: TokenType,
  email: string,
): Promise<string> {
  const ctx = await request.newContext({ baseURL: API_BASE });
  try {
    const res = await ctx.get(
      `/api/test/tokens?type=${encodeURIComponent(type)}&email=${encodeURIComponent(email)}`,
    );
    if (!res.ok()) {
      throw new Error(
        `Failed to fetch ${type} token for ${email}: ${res.status()} ${await res.text()}`,
      );
    }
    const body = (await res.json()) as { token: string };
    return body.token;
  } finally {
    await ctx.dispose();
  }
}
```

```typescript
export interface CreatedInvitation {
  id: string;
  team_id: string;
  invitee_email: string;
  token: string;
}

export async function inviteMember(input: {
  teamId: string;
  email: string;
  role?: "admin" | "editor" | "viewer";
}): Promise<CreatedInvitation> {
  const ctx = await createApiClient();
  try {
    const res = await ctx.post(`/api/teams/${input.teamId}/invitations`, {
      data: { invitee_email: input.email, role: input.role ?? "editor" },
    });
    if (!res.ok()) {
      throw new Error(`inviteMember failed: ${res.status()} ${await res.text()}`);
    }
    return (await res.json()) as CreatedInvitation;
  } finally {
    await ctx.dispose();
  }
}
```

```bash
echo "Flaky test attempts: $flaky_count" >> "$GITHUB_STEP_SUMMARY"
```
