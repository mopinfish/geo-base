**狙い**: PR ごとに API レスポンス相当箇所と MCP の公開コード表面に日本語混入がないことを Python `ast` モジュールで解析・fail-fast 検出する。

> 履歴メモ: 当初は bash + `grep -P` で実装する案だったが、macOS BSD grep が `-P` 非対応である点と、`detail=(...)`、f-string、辞書、positional 引数といった多様な構文を robust に扱うため AST 解析に切り替えた。

- [ ] **Step 1: AST 検出のターゲット確定**

スコープ:
- `api/lib/routers/**/*.py` の `Call.keywords[detail|content]` value
- `api/lib/models/**/*.py` の `Call.keywords[description]` value (Pydantic Field)
- `api/lib/routers/**/*.py` の `HTTPException(positional)[1]` / `JSONResponse(positional)[0]` / `Response(positional)[0]`
- `mcp/server.py` / `mcp/errors.py` / `mcp/tools/**/*.py` の全 `Constant(str)` (関数 docstring 含む)

スコープ外:
- Python のコメント (`# ...`) — AST に出ない
- モジュールトップの docstring — 内部説明用として残置
- README/README.ja.md / plan doc など Markdown

- [ ] **Step 2: AST スクリプトを実装**

`.github/workflows/scripts/i18n_guard.py` (Python 3.11+):

- `JP_CHARS` 正規表現で Hiragana / Katakana / CJK 統合漢字を判定
- `iter_string_literals(node)`: `ast.walk(node)` で部分木の `Constant(str)` を全て yield
- `scan_kwargs(path, kwarg_names)`: `Call.keywords[detail|description|content]` value + HTTPException/JSONResponse/Response の positional 引数を検査
- `scan_all_strings(path)`: MCP ファイル用、関数 docstring 含む全 STRING 走査 (モジュール docstring のみ除外)
- `_parse_file(path)` で OSError / UnicodeDecodeError / SyntaxError を `GuardError` に変換し、main() で集計して **parse 失敗 = exit 2** で fail-fast (silent bypass 防止)

実装の詳細は `.github/workflows/scripts/i18n_guard.py` を直接参照。

```python
# api/lib/errors.py
"""Structured API error responses (i18n Phase 2 / Issue #106)."""
from __future__ import annotations

from enum import Enum
from typing import Any, Optional

from fastapi import HTTPException


class ErrorCode(str, Enum):
    """Machine-readable error code. The UI (Phase 3, next-intl) translates by code."""

    # auth
    AUTH_INVALID_CREDENTIALS = "auth_invalid_credentials"
    AUTH_FORBIDDEN = "auth_forbidden"
    AUTH_TOKEN_EXPIRED = "auth_token_expired"
    # tileset
    TILESET_NOT_FOUND = "tileset_not_found"
    TILESET_NAME_CONFLICT = "tileset_name_conflict"
    # ... (Task 5 で実コード移行時に追加)

    INTERNAL_UNEXPECTED = "internal_unexpected"


def api_error(
    status_code: int,
    code: ErrorCode,
    message: str,
    *,
    details: Optional[dict[str, Any]] = None,
) -> HTTPException:
    """Raise/return helper for structured error responses.

    Resulting body: `{"error": {"code": ..., "message": ..., "details": ...?}}`.
    """
    payload: dict[str, Any] = {"error": {"code": code.value, "message": message}}
    if details:
        payload["error"]["details"] = details
    return HTTPException(status_code=status_code, detail=payload)
```

- [ ] **Step 1: TypeScript 型と translator**

```typescript
// app/src/lib/api-errors.ts
export interface ApiError {
  code: string;
  message: string;
  details?: Record<string, unknown>;
}

// 日本語 fallback (Phase 3 で next-intl に置き換え)
const JA_MESSAGES: Record<string, string> = {
  auth_invalid_credentials: "メールアドレスまたはパスワードが正しくありません",
  auth_forbidden: "この操作を実行する権限がありません",
  tileset_not_found: "タイルセットが見つかりません",
  // ... 段階拡張
};

export function translateApiError(
  err: ApiError,
  locale: "ja" | "en" = "ja",
): string {
  if (locale === "en") return err.message; // API は英語のため透過
  return JA_MESSAGES[err.code] ?? err.message;
}
```
