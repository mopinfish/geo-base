## Season 4 の位置づけ

Season 3 で「インフラ移行・認証・UI 品質・データ管理・エンタープライズ基盤」が整った。
Season 4 は **「外に開く」「賢くする」「耐える」** の 3 方針に沿い、
geo-base を内部管理ツールから開発者・組織・本番環境に耐えるプラットフォームへ育てる。

| 方針 | 説明 |
|------|------|
| 🌐 外に開く | 公開 API・SDK・Webhook でサードパーティ連携を可能にする |
| 🤖 賢くする | MCP / LLM との統合を深め、自然言語で地図データを操作できるようにする |
| 🛡️ 耐える | CDN・レプリカ・バックアップ・監査ログで本番運用品質を高める |

## リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| 自然言語クエリの SQL インジェクション | 高 | LLM 生成 SQL は必ずパラメータ化クエリで実行。直接文字列結合禁止 |
| Webhook 配信先への SSRF | 中 | URL バリデーションでプライベートアドレス帯（RFC 1918）を拒否 |
| バックアップの Tigris 費用増大 | 低 | 7世代ローテーション + Lifecycle Policy で古いものを自動削除 |
| CDN キャッシュとリアルタイム性のトレードオフ | 中 | タイルセット更新時に `Cache-Tag` ベースの即時パージを実施 |
| SDK 公開後の破壊的変更 | 高 | セマンティックバージョニング厳守。`/v1/` URL は廃止前に 6 ヶ月以上維持 |
| LLM API コスト（AI 機能） | 中 | リクエスト単位でキャッシュ（同一クエリの再実行を抑制） |
| SSO 実装の認証脆弱性 | 高 | OIDC / SAML ライブラリは実績あるもののみ使用。独自実装しない |
